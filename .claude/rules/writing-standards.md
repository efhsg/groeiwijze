# Writing Standards

Van toepassing bij het schrijven of bewerken van documentatie in `.claude/`.

## Denkraamwerk

| Vraag | Toepassing |
|-------|------------|
| Wie leest dit? | Claude of de ontwikkelaar die een rule of skill uitbreidt |
| Wat is het kernbericht? | Eén zin die de lezer meeneemt — niet de implementatie, tenzij nodig |
| Is de volgorde logisch? | Eerst wat het is, dan hoe het werkt, dan randgevallen |
| Gebruik ik de juiste term? | `BEM`, `WCAG`, `Duin Harmonie` — niet omschrijven tenzij in lopende tekst |
| Kan ik korter? | Eén heldere zin verslaat drie vage zinnen |

## Schrijfregels

- **Actieve stem** — "Claude controleert de kleuren", niet "De kleuren worden gecontroleerd"
- **Imperatief voor instructies** — "Lees het bestand", niet "Je kunt het bestand lezen"
- **Korte zinnen** — max ~25 woorden per zin
- **Lijsten voor stappen** — genummerd bij volgorde, bullets bij opsomming
- **Codevoorbeelden** — fenced code blocks met taaltype voor alle technische referenties
- **Eén concept per alinea** — één doel per sectie

## Consistentie

- Dezelfde term voor hetzelfde concept, overal in het document
- Domeinbegrippen zijn leidend: `BEM`, `Duin Harmonie`, `WCAG 2.1 AA`, `Libre Franklin`
- Code-termen in het Engels, lopende tekst in het Nederlands

## Principes

- Schrijf voor de lezer, niet voor jezelf
- Structuur is de helft van begrijpelijkheid
- Consistente terminologie is niet optioneel — het is de basis van vertrouwen
